# Day 13 - Mixed Array Problems

## Overview

Day 13 of the 100-Day DSA Revision Series focused on solving mixed array problems involving hashing, frequency counting, and optimization techniques. The session emphasized understanding problem patterns before implementation and practicing interview-oriented thinking.

---

## Topics Covered

* Majority Element
* Best Time to Buy and Sell Stock
* Frequency Counting using Hash Maps
* Array Traversal
* Optimization from Brute Force to Efficient Solutions
* Edge Case Handling
* Time and Space Complexity Analysis

---

## Programs Implemented

### 1. majority_element.cpp

**Concepts Used:**

* Arrays
* Hashing (`unordered_map`)
* Frequency Counting
* Conditional Checking

**Description:**
Finds the element that appears more than `n/2` times in an array. If no such element exists, the program displays an appropriate message.

**Complexity:**

* Time Complexity: O(n)
* Space Complexity: O(n)

---

### 2. best_time_to_buy_sell_stock.cpp

**Concepts Used:**

* Arrays
* Single Pass Traversal
* Tracking Minimum Value
* Profit Calculation
* Edge Case Handling

**Description:**
Determines the maximum profit that can be achieved by buying and selling a stock exactly once.

**Complexity:**

* Time Complexity: O(n)
* Space Complexity: O(1)

---

## Complexity Analysis Practiced

| Problem                         | Time Complexity | Space Complexity |
| ------------------------------- | --------------- | ---------------- |
| Majority Element                | O(n)            | O(n)             |
| Best Time to Buy and Sell Stock | O(n)            | O(1)             |

---

## Key Learnings

* Frequency-based problems can often be solved efficiently using hash maps.
* Before coding, understanding the actual problem statement is essential.
* Tracking the minimum value seen so far can optimize many array problems.
* Edge cases such as empty arrays should always be considered.
* Interview questions often test pattern recognition more than syntax knowledge.
* Optimal solutions are not always complicated; sometimes a single traversal is enough.
