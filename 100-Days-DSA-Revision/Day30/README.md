# Day 30 - Sorting Applications

## 📌 Overview

Day 30 focuses on understanding how sorting is used to solve real-world and interview problems efficiently. Instead of learning a new sorting algorithm, this day explores the practical applications of sorting and how it simplifies many complex problems.

---

## 🎯 Objectives

- Understand why sorting is used.
- Learn how sorting improves algorithm efficiency.
- Solve common problems based on sorting applications.
- Strengthen problem-solving skills using sorting as a preprocessing step.

---

## 📚 Topics Covered

- Why Sorting is Important
- How Sorting Works
- Applications of Sorting
- Remove Duplicates from a Sorted Array
- Pair with Given Sum (Two Pointer Technique)
- Frequency Count using Sorting
- Time Complexity Analysis
- Interview Insights

---

## 💻 Programs Implemented

### 1. Remove Duplicates from a Sorted Array
- Removes duplicate elements from a sorted array.
- Demonstrates how sorting places duplicate elements together.
- Time Complexity: **O(n)**

### 2. Pair with Given Sum (Two Pointer Technique)
- Finds whether a pair of elements equals the target sum.
- Uses sorting followed by the Two Pointer approach.
- Time Complexity: **O(n log n)**

### 3. Frequency Count using Sorting
- Counts the frequency of each distinct element.
- Uses sorting to group identical elements together.
- Time Complexity: **O(n log n)**

---

## 🧠 Key Concepts Learned

- Sorting is often a preprocessing step rather than the final solution.
- Many algorithms become simpler after sorting.
- Two Pointer Technique works efficiently on sorted arrays.
- Duplicate detection becomes easier after sorting.
- Frequency counting can be performed efficiently after sorting.
